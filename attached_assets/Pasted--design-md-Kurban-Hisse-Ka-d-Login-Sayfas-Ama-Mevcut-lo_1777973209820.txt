# design.md — Kurban Hisse Kağıdı Login Sayfası

## Amaç

Mevcut login sayfasını daha modern, daha premium ve daha okunabilir bir tasarıma dönüştür.  
Sayfa **karman çorman görünmemeli**, kullanıcıya güven vermeli ve tek odağı şifrelerle giriş yapma işlemi olmalı.

Bu tasarım sadece görsel süsleme için yapılmayacak. Mevcut işlev korunacak:

- Kullanıcı 2 farklı şifre girer.
- Şifreleri göster/gizle ikonları çalışır.
- `Giriş Yap` butonu mevcut login akışını tetikler.
- Başlık, açıklama ve alttaki şiir korunur.
- Gereksiz buton, link, menü, kart, istatistik, sahte özellik veya boş UI eklenmez.

---

## Genel Tasarım Dili

Tasarım dili:

- Dark theme
- Premium
- Sade
- Kurumsal
- Güven veren
- Hafif glassmorphism
- Yeşil / turkuaz vurgu renkleri
- Fazla efekt kullanmadan modern görünüm

Sayfa, dini/kurban temasıyla uyumlu şekilde ağırbaşlı ve sade görünmeli.  
Ne çok oyunlaştırılmış, ne de çok kurumsal ve soğuk olmalı.

---

## Sayfa Yapısı

Sayfa tek ekranlık bir login ekranı olmalı.

Genel yapı:

1. Tam ekran koyu arka plan
2. Ortalanmış login kartı
3. Kartın altında zarif şiir alanı
4. Mobilde aynı yapı dikey olarak korunmalı

Sayfa yatayda ve dikeyde dengeli görünmeli.  
Login kartı ekranın tam ortasına yakın durmalı, şiir karttan kopuk ama onunla aynı tasarım dilinde olmalı.

## Arka Plan

Mevcut düz lacivert arka plan yerine daha derinlikli bir arka plan kullanılmalı.

Önerilen arka plan:

```css
background:
  radial-gradient(circle at 80% 15%, rgba(0, 255, 170, 0.16), transparent 28%),
  radial-gradient(circle at 15% 85%, rgba(0, 190, 255, 0.10), transparent 32%),
  linear-gradient(135deg, #07111f 0%, #101b2c 45%, #07101c 100%);
```

Arka plan kuralları:

- Çok parlak olmamalı.
- Kullanıcıyı yormamalı.
- Form alanını öne çıkarmalı.
- Hafif glow olabilir ama abartılmamalı.
- Yıldız, partikül, grid gibi detaylar çok az ve düşük opaklıkla kullanılabilir.

İsteğe bağlı olarak `::before` veya `::after` ile yumuşak blur ışıklar eklenebilir.

## Login Kartı

Login kartı sayfanın ana odağıdır.

### Kart Stili

```css
background: rgba(8, 17, 31, 0.72);
border: 1px solid rgba(148, 163, 184, 0.22);
box-shadow:
  0 24px 80px rgba(0, 0, 0, 0.36),
  inset 0 1px 0 rgba(255, 255, 255, 0.04);

```

Desktop:

```css

padding: 48px 56px;
```

Tablet:

```css

```

Mobil:

```css

```

1. Küçük ikon/emblem alanı

3. Alt başlık

Elemanlar arasında yeterli boşluk bırakılmalı.

```css

  0 22px 44px rgba(0, 217, 111, 0.28),
  inset 0 1px 0 rgba(255, 255, 255, 0.22);
```

## Şiir Alanı

Şiir, mevcut tasarımda sayfaya karakter katıyor.  
Bu alan korunmalı ama daha zarif ve okunabilir yapılmalı.

### Yerleşim

- Login kartının altında olmalı.
- Kartla arasında `28px - 36px` boşluk olmalı.
- Ortalanmış olmalı.

```css
max-width: 520px;
```

### Dekoratif Çizgiler

Şiirin üstünde ve altında çok hafif bir çizgi/ornament olabilir.

- Şiir fazla parlak olmamalı.

## Responsive Davranış

### Desktop

- Kart ortada ve genişlik maksimum 560px.
- Başlık büyük ve güçlü.
- Inputlar geniş ve ferah.
- Sayfa yüksekliği yeterliyse login kartı tam ortaya yakın durmalı.

### Tablet

- Kart genişliği 520px civarı.
- Padding biraz azalmalı.
- Elemanlar aynı sırayı korumalı.

### Mobil

Mobilde tasarım kesinlikle taşmamalı.

```css
min-height: 100svh;
padding: 24px 16px;
```

Mobil kuralları:

- Kart genişliği `100%`.
- Başlık çok büyümemeli.
- Input yüksekliği 58px civarına düşebilir.
- Buton yüksekliği 58px olabilir.
- Şiir fontu 14px - 15px arası olabilir.
- Kart ve şiir arasında yeterli boşluk kalmalı.
- Klavye açıldığında form kullanılabilir kalmalı.

## Boşluk Sistemi

Tasarımda düzenli bir spacing sistemi kullanılmalı.

Önerilen boşluklar:

```css
--space-xs: 6px;
--space-sm: 10px;
--space-md: 16px;
--space-lg: 24px;
--space-xl: 32px;
--space-2xl: 48px;
```

Form içi:

- Label ile input arası: `8px - 10px`
- İki input grubu arası: `20px - 24px`
- Son input ile buton arası: `28px - 32px`
- Başlık alanı ile form arası: `34px - 42px`

## Etkileşimler

Sayfa hafif ama kaliteli mikro etkileşimler içermeli.

Kullanılabilecek animasyonlar:

- Kart sayfa açılırken hafif yukarıdan gelip opacity ile görünür.
- Input focus olduğunda border ve glow değişir.
- Buton hover’da hafif yükselir.
- Hata durumunda input kısa ve zarif bir şekilde vurgulanır.

Animasyonlar abartılmamalı.

```css
transition: all 160ms ease;
```

Sayfa açılış animasyonu:

```css
@keyframes cardEnter {
  from {
    opacity: 0;
    transform: translateY(14px) scale(0.985);
  }
  to {
    opacity: 1;
    transform: translateY(0) scale(1);
  }
}
```

## Hata Durumu

Eğer yanlış şifre hatası varsa tasarımda sade gösterilmeli.

Örnek:

```text
Şifreler hatalı. Lütfen tekrar deneyin.
```

Hata stili:

```css
background: rgba(239, 68, 68, 0.10);
border: 1px solid rgba(239, 68, 68, 0.28);
color: #fecaca;
border-radius: 12px;
padding: 12px 14px;
font-size: 14px;
```

Hata mesajı butonun üstünde veya formun altında olabilir.  
Sayfayı bozacak büyük hata kutuları kullanılmamalı.

## Erişilebilirlik

Tasarım görsel olarak güzel olduğu kadar kullanılabilir de olmalı.

Kurallar:

- Her input için label kullanılmalı.
- Şifre göster/gizle butonlarında `aria-label` bulunmalı.
- Focus state net görünmeli.
- Renk kontrastı yeterli olmalı.
- Form klavye ile kullanılabilir olmalı.
- Enter tuşu ile giriş yapılabilmeli.
- Sadece renk ile hata anlatılmamalı; hata metni de gösterilmeli.

Örnek aria-label:

```html
<button aria-label="1. şifreyi göster veya gizle">
```
